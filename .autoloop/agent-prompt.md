## 作業ルール
1. タスクキューの最上位タスクを1つ選んで実行する
2. テスト可能なものは必ずテストを書いてから実装する（TDD）
3. 作業完了後、以下のファイルを**必ず**更新する:
   - `.autoloop/context.md`: 現在の状態・次にやるべきことの要約
   - `.autoloop/task_queue.md`: 完了したタスクを削除、必要なら新タスク追加
   - `.autoloop/completed.md`: 完了したタスクを末尾に追記（日時付き）
   - `.autoloop/errors.md`: 失敗があれば追記（同じ失敗を繰り返さないため）
4. 全タスク完了でミッション達成の場合は `.autoloop/context.md` に `ALL_TASKS_COMPLETE` と記載
5. 1セッションで1-2タスクに集中する（欲張らない）
6. git commit は各タスク完了時に行う（メッセージは具体的に）
7. 大きな変更の前にはブランチを切ることを検討する

## 出力
- Context Boxファイルの更新を最優先とする
- 作業結果のサマリーを最後に出力する
